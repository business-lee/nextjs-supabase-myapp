## 1. 문제 (Problem)

소규모 동호회 주최자가 겪는 3가지 핵심 문제:

1. **모임 공지 분산**: 카카오톡 단체방, 문자, 메모장 등 여러 채널에 공지가 흩어져 참여자가 최신 정보를 놓침
2. **참여자 관리 불편**: 신청 수기 관리(스프레드시트, 채팅방 댓글)로 인한 실수, 대기자 순번 추적 어려움
3. **정산 번거로움**: 카풀 조율·비용 정산을 채팅방에서 일일이 수동으로 처리해야 하는 비효율

### 기존 대안

- 카카오톡 단체방 (공지·정산 혼재, 히스토리 유실)
- 네이버 밴드 (무겁고 동호회 특화 기능 부재)
- 스프레드시트 (참여자 접근 불편, 실시간성 없음)

---

## 2. 고객 세그먼트 (Customer Segments)

### 주요 고객: 소규모 동호회 주최자 (Host)

- 10~30명 규모 동호회를 혼자 운영하는 개인
- 수영, 헬스, 등산, 보드게임 등 취미 동호회 운영자
- 정기·비정기 모임을 월 1~4회 개최하는 사람

### 보조 고객: 동호회 회원 (Member)

- 초대 링크로 접근하는 일반 참여자
- 모바일 중심 이용 환경

### 얼리어답터

- IT 종사자 또는 디지털 도구에 익숙한 취미 동호회 운영자
- 현재 스프레드시트나 채팅방으로 관리에 불편함을 느끼는 주최자

---

## 3. 가치 제안 (Unique Value Proposition)

**"모임 공지부터 정산까지, 링크 하나로 끝내는 동호회 운영 도구"**

- 초대 링크 하나로 참여 신청·카풀·정산을 한 페이지에서 처리
- 주최자는 승인/거절·대기자 순번 관리를 자동화로 해방
- 참여자는 앱 설치 없이 링크만으로 모든 정보 확인 및 신청 가능

---

## 4. 해결책 (Solution)

| 문제             | 해결책                                                |
| ---------------- | ----------------------------------------------------- |
| 공지 분산        | 모임별 공지사항 탭 (고정 공지, CRUD)                  |
| 참여자 관리 불편 | 자동/수동 승인, 대기자 순번 자동 관리, 초대 링크 공유 |
| 카풀 조율 어려움 | 드라이버 등록 → 동승 신청 → 수락/거절 흐름 내재화     |
| 정산 번거로움    | 1/N 자동 계산, 납부 현황 체크, 정산 링크 공유         |

---

## 5. 경쟁우위 (Unfair Advantage)

- **Zero friction 온보딩**: 회원이 앱 설치·회원가입 없이 초대 링크 접속만으로 모임 정보 확인 가능
- **역할별 UX 분기**: 동일 URL에서 주최자(관리)와 참여자(조회)가 최적화된 화면 제공
- **RLS 기반 보안**: Supabase Row Level Security로 데이터 무단 접근 원천 차단
- **모바일 퍼스트**: 동호회원의 주요 접근 환경인 모바일 최적화 레이아웃

---

## 6. 수익 흐름 (Revenue Stream)

MVP 단계 (현재):

- **무료 제공** — 사용자 확보 및 제품-시장 적합성(PMF) 검증 우선

향후 수익화 옵션:

- **프리미엄 플랜**: 모임 수 제한 해제, 참여자 수 확장, 고급 정산 기능
- **동호회 팀 플랜**: 다중 주최자 협업, 반복 모임 템플릿
- **광고/제휴**: 동호회 활동 관련 용품·장소 제휴 (선택적)

## 7. 비용 구조 (Cost Structure)

| 항목   | 내용                                                                  |
| ------ | --------------------------------------------------------------------- |

| 도메인 | 커스텀 도메인 연간 비용 (~$15/년)                                     |
| 마케팅 | 커뮤니티 입소문 중심 (초기 무비용)                                    |

---

## 8. 주요 지표 (Key Metrics)

| 지표                  | 측정 방법                        |
| --------------------- | -------------------------------- |
| 주최자 활성화율       | 가입 후 첫 모임 생성 완료 비율   |
| 초대 링크 전환율      | 링크 접속 → 참가 신청 완료 비율  |
| 모임당 평균 참여자 수 | 승인된 참여자 수 / 전체 모임 수  |
| 정산 기능 사용률      | 모임 중 정산 탭 입력 완료 비율   |
| 재방문율              | 주최자가 2개 이상 모임 생성 비율 |

### 성공 기준 (MVP)

- 주최자: 로그인 후 5분 이내 모임 생성 + 초대 링크 발급
- 참여자: 링크 접속 → 로그인 → 신청 3단계 이내 완료
- 모바일 360px 이상에서 모든 주요 페이지 정상 표시

---

## 9. 채널 (Channels)

- **카카오톡 공유**: 초대 링크를 카카오톡으로 직접 공유 (핵심 바이럴 채널)
- **커뮤니티**: 동호회 관련 온라인 커뮤니티 (네이버 카페, 오픈채팅)
- **입소문**: 주최자 → 타 주최자 추천 (네트워크 효과)
- **SEO**: 동호회 관리, 모임 정산 관련 검색 유입 (장기)
